# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT create a public issue** for security vulnerabilities
2. **Email the maintainers** privately with details
3. **Provide clear steps to reproduce** the vulnerability
4. **Include potential impact assessment**

We will:
- **Acknowledge your report** within 48 hours
- **Provide regular updates** on our progress
- **Credit you** in the security advisory (unless you prefer to remain anonymous)
- **Release a fix** as quickly as possible

## Security Best Practices

When deploying Ausgleichsliste:

- **Change the default admin password** in `appsettings.json`
- **Use HTTPS** in production environments
- **Keep the application updated** to the latest version
- **Review logs regularly** for suspicious activity
- **Backup your database** regularly

Thank you for helping keep Ausgleichsliste secure!
